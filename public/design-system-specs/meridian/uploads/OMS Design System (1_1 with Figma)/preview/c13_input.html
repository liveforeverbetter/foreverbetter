Input

Inputs allow users to enter and edit data within forms. They support multiple states—default, active, filled, error, and disabled—to provide clear feedback throughout the interaction. Use inputs for structured data entry such as emails, names, or codes. Always include a clear label, optional helper text for guidance, and error messaging when validation fails. Keep error messages specific and actionable, and use disabled states only when input is temporarily unavailable.
